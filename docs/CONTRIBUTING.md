# Contributing to QEncode

Thank you for your interest in QEncode.

## How to get involved

The fastest way to get involved is to apply for early access as a benchmark partner:  
-> [qencode-benchmark.org/apply](https://www.qencode-benchmark.org/apply)

## Reporting issues

If you find a bug or inconsistency in the benchmark specification, leaderboard data,
or documentation, please open a GitHub Issue with:

- A clear description of the problem
- The file or data entry affected
- What you expected vs what you observed

## Suggesting molecules or benchmark configurations

New molecule additions are reviewed on a quarterly basis for Suite v2.1 and beyond.
To propose a new molecule, open a GitHub Issue with the label `molecule-request`
and include: molecule name, qubit count estimate, scientific motivation, and relevant references.

## Contact

For partnership or certification enquiries: support@qencode-benchmark.org
